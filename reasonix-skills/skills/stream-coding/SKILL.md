# Stream Coding — 文档优先开发

**核心理念：** "当代码失败时，修复 spec——不是代码。"

Stream Coding 是文档优先的方法论，让 AI 生成的代码具有确定性。

## 时间分配

- **40%** 战略思考（Phase 1）— 写代码前解决难题
- **40%** AI 就绪文档（Phase 2）— Spec 写完整到 AI 零决策
- **5%** 对抗审查（Phase 2.5）— 不同 AI 攻击 spec
- **10%** 执行（Phase 3）— 代码自动流出
- **5%** 质量（Phase 4）— 测试和验证

## 五个阶段

### Phase 1: 战略思考
回答 7 个问题：
1. 问题是什么？
2. 谁在用？
3. 成功标准是什么？
4. 有什么约束？
5. 有什么替代方案？
6. 最大风险是什么？
7. 最小可行方案是什么？

### Phase 2: AI 就绪文档
写确定性 spec：
- 每个功能的精确行为
- 输入/输出类型
- 错误语义
- 边界情况
- **零歧义** — 两个开发者读同一份 spec 应该写出相同代码

### Phase 2.5: 对抗审查
用不同 AI 攻击 spec，找到漏洞，修复后再写代码。

### Phase 3: 执行
Spec → 代码。AI 按 spec 生成，不做决策。

### Phase 4: 质量
- 验证 spec 中的每个需求
- 运行测试
- 手动抽查

## 红线

**绝不在 spec 未完成时写代码。**
**绝不手动修改代码而不更新 spec。**（会产生 Divergence — 技术债务）
